# Star-HA-Web-Platform 项目目标

## 1. 项目概述

Star-HA-Web-Platform 是一个基于 Docker 的高可用 Web 集群演示平台，旨在展示企业级 Web 服务架构的核心组件和最佳实践。

## 2. 核心目标

### 2.1 高可用性 (High Availability)
- **目标**：确保服务持续可用，单点故障不影响整体服务
- **实现方式**：
  - 主备负载均衡器 (Nginx + Keepalived)
  - 多后端 Web 服务器
  - 自动故障检测与切换
  - 容器自动重启机制

### 2.2 负载均衡 (Load Balancing)
- **目标**：合理分配请求流量，提升服务性能
- **实现方式**：
  - Nginx 反向代理
  - 轮询调度算法
  - 健康检查机制
  - 会话保持支持

### 2.3 共享存储 (Shared Storage)
- **目标**：实现多节点数据一致性，简化内容管理
- **实现方式**：
  - Docker Volume 共享存储卷
  - 所有 Web 节点共享同一数据源
  - 内容更新即时生效，无需逐节点同步
  - FileBrowser 提供可视化管理界面

### 2.4 全方位监控 (Comprehensive Monitoring)
- **目标**：实时掌握系统运行状态，快速定位问题
- **实现方式**：
  - Prometheus 指标采集
  - Grafana 可视化仪表盘
  - Node Exporter 系统指标
  - 自定义告警规则

## 4. 组件说明

| 组件 | 版本 | 功能 | 端口 |
|------|------|------|------|
| nginx-lb-master | custom | 主负载均衡器 | 80, 443 |
| nginx-lb-backup | custom | 备负载均衡器 | 8080 |
| webapp1/2 | nginx:alpine | Web 应用服务器 | - |
| filebrowser | latest | 共享存储文件管理 | 8082 |
| prometheus | latest | 监控数据采集 | 9090 |
| grafana | 11.0.0 | 监控可视化 | 3001 |
| node-exporter | latest | 系统指标采集 | 9100 |

## 5. 技术栈

- **容器编排**: Docker Compose
- **负载均衡**: Nginx + Keepalived
- **Web 服务**: Nginx (Alpine)
- **共享存储**: Docker Volume + FileBrowser
- **监控**: Prometheus + Grafana
- **指标采集**: Node Exporter

## 6. 快速开始

### 6.1 环境要求
- Docker Desktop (Windows/Mac) 或 Docker Engine (Linux)
- Docker Compose v2.x
- 至少 8GB 内存
- 至少 4 核 CPU

### 6.2 启动服务
```bash
# 进入项目目录
cd Star-HA-Web-Platform

# 启动所有服务
docker compose up -d

# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f
```

### 6.3 访问服务
- Web 服务 (主): http://localhost
- Web 服务 (备): http://localhost:8080
- Prometheus: http://localhost:9090
- Grafana: http://localhost:3001 (admin/admin123)
- FileBrowser: http://localhost:8082 (无需登录)

## 8. 演示场景

### 8.1 高可用演示
1. 停止 webapp1 容器
2. 验证服务仍可通过 webapp2 访问
3. 恢复 webapp1

2. 观察响应中的服务器标识变化

### 8.3 共享存储演示
1. 通过 FileBrowser 修改文件
2. 刷新 Web 页面验证变更即时生效

### 8.4 监控演示
1. 查看 Grafana 仪表盘
2. 观察实时监控数据

## 9. 生产环境建议

1. **安全性**
   - 修改所有默认密码
   - 启用 HTTPS
   - 配置防火墙规则
   - 定期更新镜像

2. **高可用**
   - 部署到多台物理服务器
   - 使用外部负载均衡器
   - 配置数据备份

3. **监控**
   - 配置告警通知
   - 设置合理的告警阈值
   - 定期检查监控数据

## 10. 故障排查

### 10.1 容器无法启动
```bash
# 查看容器日志
docker compose logs <container_name>

# 检查端口占用
netstat -tlnp | grep <port>
```

### 10.2 服务无法访问
```bash
# 检查容器状态
docker compose ps

# 检查网络
docker network inspect star-ha-web-platform_ha-network
```
