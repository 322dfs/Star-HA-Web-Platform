# Star-HA-Web-Platform 测试文档

## 1. 测试概述

本文档定义了 Star-HA-Web-Platform 高可用 Web 集群的测试流程、测试用例和验收标准。

## 2. 测试环境

### 2.1 硬件要求

- CPU: 4 核及以上
- 内存: 8GB 及以上
- 磁盘: 20GB 可用空间

### 2.2 软件要求

- 操作系统: Windows 10/11 (WSL2) 或 Linux
- Docker Desktop: 最新版本
- Docker Compose: v2.x

### 2.3 网络配置

- 集群网络: 172.28.0.0/16
- 外部访问端口: 80, 8080, 9090, 3001, 9100, 8082

## 3. 测试分类

### 3.1 功能测试 (Functional Testing)

#### 测试用例 F-01: 服务启动验证

| 项目   | 内容                                                    |
| ---- | ----------------------------------------------------- |
| 测试目标 | 验证所有容器正常启动                                            |
| 前置条件 | Docker 环境已就绪                                          |
| 测试步骤 | 1. 执行 `docker compose up -d`2. 执行 `docker compose ps` |
| 预期结果 | 所有容器状态为 `Up` 或 `running`                              |
| 测试命令 | `docker compose ps`                                   |

#### 测试用例 F-02: Web 服务可访问性

| 项目   | 内容                                                          |
| ---- | ----------------------------------------------------------- |
| 测试目标 | 验证 Web 服务正常响应                                               |
| 前置条件 | 所有容器正常运行                                                    |
| 测试步骤 | 1. 浏览器访问 <http://localhost>2. 浏览器访问 <http://localhost:8080> |
| 预期结果 | 显示 Star-HA Web Platform 欢迎页面                                |
| 测试命令 | `curl http://localhost`                                     |

#### 测试用例 F-03: 负载均衡验证

| 项目   | 内容                                                | <br />                |
| ---- | ------------------------------------------------- | :-------------------- |
| 测试目标 | 验证请求被分发到不同后端                                      | <br />                |
| 前置条件 | Web 服务正常                                          | <br />                |
| 测试步骤 | 多次刷新页面或发送请求                                       | <br />                |
| 预期结果 | 响应中显示不同的后端服务器标识                                   | <br />                |
| 测试命令 | \`for i in {1..10}; do curl -s <http://localhost> | grep "Server"; done\` |

| 项目   | 内容                                                           |

### 3.2 高可用测试 (High Availability Testing)

#### 测试用例 HA-01: 后端服务故障切换

| 项目   | 内容                                                                    |
| ---- | --------------------------------------------------------------------- |
| 测试目标 | 验证单后端故障时服务继续可用                                                        |
| 前置条件 | 双后端正常运行                                                               |
| 测试步骤 | 1. 停止 webapp1: `docker compose stop webapp1`2. 访问 Web 服务3. 恢复 webapp1 |
| 预期结果 | 服务持续可用，无中断                                                            |
| 测试命令 | `docker compose stop webapp1 && curl http://localhost`                |

#### 测试用例 HA-02: 主负载均衡器故障

| 项目   | 内容                                                                  |
| ---- | ------------------------------------------------------------------- |
| 测试目标 | 验证主 LB 故障时的处理                                                       |
| 前置条件 | 主备 LB 正常运行                                                          |
| 测试步骤 | 1. 停止 nginx-lb-master2. 通过备用 LB (8080) 访问服务                         |
| 预期结果 | 备用 LB 正常提供服务                                                        |
| 测试命令 | `docker compose stop nginx-lb-master && curl http://localhost:8080` |

#### 测试用例 HA-03: 服务自动恢复

| 项目   | 内容                                            |
| ---- | --------------------------------------------- |
| 测试目标 | 验证容器异常退出后自动重启                                 |
| 前置条件 | 容器配置 restart: unless-stopped                  |
| 测试步骤 | 1. 强制停止容器2. 等待自动重启                            |
| 预期结果 | 容器自动重启并恢复正常                                   |
| 测试命令 | `docker kill webapp1 && sleep 5 && docker ps` |

## 5. 快速测试脚本

### 5.1 一键功能测试

```bash
#!/bin/bash
echo "=== Star-HA-Web-Platform 功能测试 ==="

echo "[1/7] 检查容器状态..."
docker compose ps

echo "[2/7] 测试主负载均衡器..."
curl -s -o /dev/null -w "HTTP Status: %{http_code}\n" http://localhost

echo "[3/7] 测试备用负载均衡器..."
curl -s -o /dev/null -w "HTTP Status: %{http_code}\n" http://localhost:8080

echo "[4/7] 测试 FileBrowser..."
curl -s -o /dev/null -w "HTTP Status: %{http_code}\n" http://localhost:8082

echo "[5/7] 测试 Prometheus..."
curl -s -o /dev/null -w "HTTP Status: %{http_code}\n" http://localhost:9090

echo "[6/6] 测试 Grafana..."
curl -s -o /dev/null -w "HTTP Status: %{http_code}\n" http://localhost:3001

echo "=== 测试完成 ==="
```

### 5.2 故障切换测试

```bash
#!/bin/bash
echo "=== 故障切换测试 ==="

echo "[1/3] 停止 webapp1..."
docker compose stop webapp1
sleep 2

echo "[2/3] 测试服务可用性..."
curl -s http://localhost | head -5

echo "[3/3] 恢复 webapp1..."
docker compose start webapp1

echo "=== 测试完成 ==="
```

## 8. 视频演示要点

### 8.1 演示顺序建议

1. **项目介绍** (2分钟)
   - 架构概览
   - 核心功能说明
2. **服务启动演示** (3分钟)
   - Docker Compose 启动
   - 容器状态检查
3. **Web 服务演示** (3分钟)
   - 访问主/备负载均衡器
   - 负载均衡效果展示
4. **共享存储演示** (2分钟)
   - 通过 FileBrowser 修改内容
   - 即时生效验证
5. **监控演示** (3分钟)
   - Grafana 仪表盘
   - 实时数据展示
6. **故障切换演示** (3分钟)
   - 停止后端服务
   - 验证服务持续可用
   - 恢复服务
7. **总结** (2分钟)
   - 项目亮点
   - 适用场景

### 8.2 演示注意事项

- 提前启动所有服务，确保稳定
- 准备好各服务账号密码
- 故障切换演示前确认服务正常
- FileBrowser 无需登录即可使用
